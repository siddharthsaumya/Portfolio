What to learn after HTML and CSS?

For few it can be difficult to choose, what to learn after learning both HTML and CSS. As there are plethora of courses to learn. This could be a guide to them.

I am assuming that you have already learned HTML and CSS. If you haven't you should learn HTML and CSS first. There is a never-ending list of things you have to learn. But here we have a short suggestion of what you should learn to become a good web developer.

So, Lets start out list:-

  1. Javascript

    2. JavaScript aka JS aka Vanilla JavaScript, it is most well-known as the scripting language for Web pages. To become at least a good front end web developer you need to learn it, properly. It is used to make a static webpage function as coded in js. It is used in both front-end and back-end. So, first, you should shoot for js.

  2. jQuery

    3. jQuery is a JavaScript library designed to simplify HTML DOM tree traversal and manipulation, as well as event handling, CSS animation, and Ajax. It is free, open-source software using the permissive MIT License. As of May 2019, jQuery is used by 73% of the 10 million most popular websites.

  3. Bootstrap

    4. Bootstrap is a free and open-source CSS framework. It contains CSS- and JavaScript-based design templates for typography, forms, buttons, navigation, and other interface components. It simply makes designing and making websites a lot easier. I usually use it for small paychecksüòù. You can use it to design a good looking website very quickly.

  4. React

    5. React is a JavaScript library for building user interfaces. It is maintained by Facebook and a community of individual developers and companies. Helps to make beautiful UX and UI.

  5. Unix command lines

    6. Unix command lines are used during working on back-end servers. Its a must. I wont tell you about it much, i want you to go on youtube and search what it is and give yourself some worküòÅ.

  6. Node.js / PHP

    7. Node.js is an open-source server environment. A common task for a web server can be to open a file on the server and return the content to the client. Basically it is a server-side language that uses javascript to operate.

    PHP is an acronym for "PHP: Hypertext Preprocessor". PHP is a widely-used, open-source scripting language. PHP scripts are executed on the server-side.

    Both PHP and Node.js have the same function. They are just two other ways to do the work.


    To see a detailed comparison CLICK HERE.

  7. Express

    8. Express is a minimal and flexible Node.js web application framework that provides a robust set of features for web and mobile applications. Just like jQuery was for javascript.

  8. XML / JSON

    9. They both are data-interchange format. They are human-readable and writable. Used to exchange data between browser and server.

    XML: Extensible markup language. XML was designed to be self-descriptive. Its is an older way but still used and has a larger community base.

    JSON: JavaScript Object Notation. Its newer way but its community is not that much developed, though companies these days and most of the startups are using it, instead of XML.


    To see a detailed comparison CLICK HERE.

  9. EJS

    10. Embedded JavaScript. EJS is a simple templating language that lets you generate HTML markup with plain JavaScript. It makes working with multiple pages websites and contents a lot easier. And saves your fingers from carpal tunnel syndrome üòÇ.

  10. Database (SQL / NoSQL)

    11. A database is a collection of information that is organized so that it can be easily accessed, managed and updated. There are basically two types of database, SQL and NoSQL.

    SQL: Structured Query Language. It uses tabular form to store data, which you can easily access in excel sheets. In short, all data that is been used, by SQL that data is being saved to your own computer. So all that data is taking storage space on your own computer. Some examples are MS Access, SQL Server, MySQL.

    NoSQL: no SQL. In a NoSQL database, a book record is usually stored as a JSON document. These data are not arranged and can create confusion for who don't know JSON.
